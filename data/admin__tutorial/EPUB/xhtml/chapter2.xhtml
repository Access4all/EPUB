2. Links, images and multimedia

Up to now, we did about heading structure and lists. Having a correct heading structure throughout the whole book is the most important, but a little boring, isn't it ? In this second chapter, we will first learn how to create links, and then how to embed illustrations and videos in our documents.

2.1. Making links

Making links is also very important. Being able to link documents with eachother is the greatest power of the web and digital books against good old paper books. Let's say that, at this point, we want to remind you about headings structure discussed in previous chapter. We are going to make an internal link, internal because it points to a document within the same book. You have several methods to create one such link. First of all, find the section you want to link to in the table of contents on the left panel, then:

  • Wheither you drag and drop it onto the edition area
  • or you click with the right mouse button to open a context menu, choose the Copy address option, highlight the text you want to turn into a link in the edition area, click on the link button or press Ctrl+K, paste the URL previously copied in the URL field of the dialog box and press OK.
  • Paste the obtained URL from the Copy address command directly into the edition area at the desired place instead of passing through the dialog box.

The second method has the advantage to avoid having to modify the link text afterwards, while the other two are probably a bit faster. Try experimenting with more than a single methods and link to chapter 1.1!

We also want to add a link to our old friend, Google. This is an external link, meaning that it points to something residing outside of the book; a network connection will be required to visit the link. To do it, highlight the word Google, click on the link button on the toolbar, and manually enter its address on the URL field of the dialog box

You can always see or modify the URL to which a link points to by choosing the Link properties option in the context menu, which you will obtain by cliking with the right mouse button on the link text.

2.2. Illustrations

We will now talk about illustrations. There are multiple ways to add one in your document :

  • Copy an image file from your hard disk and paste it directly in the edition area
  • Drag an image from your hard disk and drop in onto the edition area
  • Copy the URL of any image found on the Internet and paste it in the edition area
  • Find an image which is already Inside your book with the Files tab, and drag and drop it onto the edition area.
  • Click on the Illustration button in the toolbar and fill in the URL or paste and URL manually

Note that all methods don't work with all browsers. All these methods make a dialog box appear, where you can put the URL of the illustration (automatically filled if you are using drag and drop or paste), a legend which will be placed Under the image, and an alternative text only visible for screen reader users and useful to describe the image if the legend isn't sufficient. The legend is visible by anyone. A special checkbox let you set the image as being illustrative only. This later checkbox must be checked if the image isn't relevant for screen reader users, that is, if the image don't give any useful information to understand the rest of the document. A style combobox will also allow you to choose if the illustration should take the entire width, or if it has to float left or right with a given size.

Try to add illustrations here, using a pair of the methods explained above!

2.3. Icons

Adding an icon is again allmost the same as adding an illustration, except that you must go by the Insert icon dialog box. Paste as well as drag and drop always insert an illustration or a multimedia clip, never an icon. There is no legend, only an alternative text. Icons are used to put images directly Inside the text, while illustrations make a separate independant block. We know that you ar happy, you may insert a few smileys here to show it!

2.4. Multimedia clips

Adding a multimedia clip is allmost the same as adding an illustration. You can use allmost the same methods as described above. When pasting or drag-and-dropping, the fact that you are trying to insert a video or an audio clip and not an image is automatically deduced from file extension.

URL pointing to youtube are also recognized and their insertion cause a frame to appear instead of an integrated player. IN that case a network connection is of course required to Watch the video. Try to put here some youtube videos !
